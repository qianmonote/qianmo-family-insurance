## 1. 项目背景

随着用户在产品中沉淀的个人数据（家庭成员信息、保单、总结记录等）增多，需要建立用户账号体系以保障数据归属与隐私安全。同时，为了提升用户日常获取信息的效率，新增"链接一键总结"功能：用户粘贴一个外部链接（公众号文章、小红书图文/视频、抖音、B站视频等），系统自动抓取内容并通过 AI 生成结构化总结，帮助用户快速获取核心信息。

### 1.1 平台范围说明

本文档为**第一期需求**，产品形态**仅支持 PC 端网页**（不做移动端 H5 / App / 响应式适配），原因是目标用户的核心使用场景（整理资料、撰写总结）在 PC 端的操作效率最高。后续如有移动端需求，将在下一期单独评估和规划。

## 2. 产品目标

| 目标类型 | 描述 |
|---|---|
| 业务目标 | 建立用户账号体系，为后续个性化功能（家庭成员管理、保单管理、个人总结记录等）打下基础 |
| 用户目标 | 用户可以通过登录保护自己的数据；可以快速将看到的图文/视频内容转化为结构化总结，节省阅读/观看时间 |
| 衡量指标 | 1）登录成功率 ≥ 99%；2）链接总结任务成功率 ≥ 90%；3）图文总结平均响应时间 ≤ 30s |

## 3. 用户角色与使用场景

### 3.1 用户角色

- **C 端家庭用户（主账号）**：登录后使用全部功能，包括链接总结。

### 3.2 用户故事

| 编号 | 用户故事 |
|---|---|
| US-1 | 作为一个用户，我希望注册并登录账号，以便我的数据只有我自己能访问 |
| US-2 | 作为一个已登录用户，我希望未登录时无法访问任何功能页面，保证数据安全 |
| US-3 | 作为一个用户，我希望粘贴一篇公众号文章链接，快速获得这篇文章的内容总结 |
| US-4 | 作为一个用户，我希望粘贴一个抖音/B站视频链接，快速获得视频内容的总结，而不用看完整个视频 |
| US-5 | 作为一个用户，我希望在提交链接时补充我的具体要求（如"帮我整理理财相关的内容"），让总结更贴合我的需求 |
| US-6 | 作为一个用户，我希望提交长视频总结任务后可以离开页面做别的事，处理完成后能在记录里看到结果 |
| US-7 | 作为一个用户，我希望查看历史总结记录，方便随时回顾 |

## 4. 功能需求

### 4.1 功能架构总览

```
阡陌家庭保
├── 登录模块
│   ├── 注册
│   ├── 登录
│   ├── 登出
│   └── 路由保护
└── 链接一键总结
    ├── 链接输入与识别
    ├── 补充要求输入
    ├── 内容抓取（图文 / 视频）
    ├── AI 总结生成
    ├── 异步任务处理（长视频）
    └── 总结记录列表
```

### 4.2 模块一：登录模块

#### 4.2.1 功能说明

应用所有功能均需登录后才能使用。未登录用户访问任何页面将被重定向至登录页；登录/注册成功后建立会话，会话过期或主动登出后需重新登录。

#### 4.2.2 功能需求列表

| 编号 | 需求点 | 详细描述 | 优先级 |
|---|---|---|---|
| L-1 | 用户注册 | 邮箱 + 密码注册；密码长度 ≥ 8 位；邮箱已注册时提示"该邮箱已注册" | P0 |
| L-2 | 用户登录 | 邮箱 + 密码登录；成功后创建会话并跳转至首页 | P0 |
| L-3 | 登录态保持 | 登录态默认保持 7 天，过期后需重新登录 | P0 |
| L-4 | 登出 | 用户主动登出后清除会话，跳转至登录页 | P0 |
| L-5 | 全局路由保护 | 未登录访问任意受保护页面 → 重定向至登录页；登录成功后跳转回原目标页 | P0 |
| L-6 | 错误提示 | 账号不存在/密码错误等场景给出统一、不暴露敏感信息的错误文案 | P1 |
| L-7 | 手机号+验证码登录 | 本期不实现，预留接口与数据字段扩展位 | 暂不开发 |

#### 4.2.3 业务流程图（文字描述）

```
[访问任意页面]
      │
      ▼
 是否已登录？──否──▶ 跳转登录页 ──▶ 登录/注册成功 ──▶ 跳转回原目标页
      │
     是
      ▼
   正常访问页面
```

#### 4.2.4 页面/原型说明

| 页面 | 说明 |
|---|---|
| 登录页 | 邮箱、密码输入框；"登录"按钮；"没有账号？去注册"入口 |
| 注册页 | 邮箱、密码、确认密码输入框；"注册"按钮；表单校验错误就地提示 |
| 全局 | 已登录用户右上角展示账号入口，提供"登出"操作 |

### 4.3 模块二：链接一键总结

#### 4.3.1 功能说明

用户在"一键总结"入口粘贴一个外部链接，系统识别链接类型、抓取内容，结合用户补充的要求调用 AI 生成结构化总结。图文类内容同步处理并快速返回；视频类内容（尤其长视频）转入后台异步处理，用户可离开页面，处理完成后在记录列表中查看结果。

#### 4.3.2 支持的链接类型

| 类型 | 范围 | 抓取内容 |
|---|---|---|
| 图文链接 | 微信公众号文章、小红书图文、公开免费网站文章 | 正文文本 + 最多 10 张配图 |
| 视频链接 | 抖音短视频、B站视频、小红书视频 | 视频标题、字幕/转写文本等 |

> 不在以上范围内的链接，提示"暂不支持该链接类型"。

#### 4.3.3 功能需求列表

| 编号 | 需求点 | 详细描述 | 优先级 |
|---|---|---|---|
| S-1 | 链接输入 | 提供输入框及"粘贴链接"按钮，支持从系统剪贴板读取链接 | P0 |
| S-2 | 链接类型识别 | 根据 URL 特征自动识别图文/视频及具体平台；无法识别时提示不支持 | P0 |
| S-3 | 补充要求输入 | 链接下方提供选填文本框，用户可填写定制化总结要求，作为 AI 提示词的补充上下文 | P1 |
| S-4 | 图文内容抓取 | 抓取文章正文文本及最多 10 张配图 | P0 |
| S-5 | 视频内容抓取 | 提取视频标题、字幕/转写等可用文本信息 | P0 |
| S-6 | AI 总结生成 | 基于抓取内容（及用户补充要求）调用 AI 生成结构化总结（要点列表/关键信息） | P0 |
| S-7 | 异步任务处理 | 视频（尤其长视频）转入后台队列处理；前端通过轮询/订阅获取任务状态；用户可离开当前页面 | P0 |
| S-8 | 任务状态展示 | 记录列表中展示"处理中/成功/失败"状态；失败任务提供"重试"按钮 | P0 |
| S-9 | 总结记录列表 | 保存用户所有总结记录，按时间倒序展示，可点击查看详情 | P0 |
| S-10 | 异常处理 | 链接失效、内容抓取失败、AI 调用失败等场景给出友好提示，不影响其他功能使用 | P1 |

#### 4.3.4 业务流程图（文字描述）

```
[用户粘贴链接]
      │
      ▼
 识别链接类型 ──不支持──▶ 提示"暂不支持该链接类型"
      │
   支持
      ▼
 用户填写补充要求（选填）
      │
      ▼
   提交任务
      │
      ├─ 图文链接 ──▶ 同步抓取 + AI总结 ──▶ 展示结果 ──▶ 存入记录列表
      │
      └─ 视频链接 ──▶ 创建后台异步任务 ──▶ 用户可离开页面
                              │
                              ▼
                      任务完成（成功/失败）
                              │
                              ▼
                      记录列表更新状态，用户可查看结果或重试
```

#### 4.3.5 页面/原型说明

| 页面/区域 | 说明 |
|---|---|
| 一键总结入口 | 输入框 + "粘贴链接"按钮 + 补充要求文本框（选填）+ "开始总结"按钮 |
| 处理中提示 | 提交后显示处理状态提示；视频类任务提示"可关闭页面，处理完成后将在记录中通知" |
| 总结结果页 | 展示来源链接、链接类型、总结内容（结构化要点）、生成时间 |
| 总结记录列表 | 列表展示所有历史任务：来源链接缩略信息、类型、状态（处理中/成功/失败）、创建时间；失败项展示"重试"按钮 |

#### 4.3.6 数据需求

新增实体 `LinkSummary`（归属于 `User`）：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | string/uuid | 主键 |
| userId | string | 关联用户，外键 |
| sourceUrl | string | 原始链接 |
| sourceType | enum | 链接类型：公众号文章 / 小红书图文 / 小红书视频 / 抖音视频 / B站视频 / 公开网站文章 |
| userPrompt | string \| null | 用户补充要求（选填） |
| status | enum | pending / processing / success / failed |
| summaryContent | text \| null | AI 生成的总结内容 |
| errorMessage | string \| null | 失败原因（失败时记录，便于排查与展示） |
| createdAt | timestamp | 创建时间 |
| updatedAt | timestamp | 更新时间 |

## 5. 非功能需求

| 类型 | 要求 |
|---|---|
| 性能 | 图文链接总结请求应在 30 秒内返回结果或进入处理中状态；视频任务后台处理不阻塞用户其他操作 |
| 安全 | 所有接口需登录鉴权；链接抓取需做 URL 校验，防范 SSRF 等风险；用户仅能访问自己的总结记录 |
| 可用性 | 各类异常（链接失效、抓取失败、AI 调用失败）均有明确提示文案，不导致页面崩溃或白屏 |
| 可扩展性 | 链接类型识别与内容抓取逻辑模块化设计，便于后续新增支持的平台 |
| 平台范围 | 本期仅交付 PC 端网页，页面布局以桌面分辨率（≥ 1280px）为基准设计，不做移动端响应式适配 |

## 6. 暂不包含范围（Out of Scope）

- 移动端 H5 / App / 小程序及响应式适配（仅做 PC 网页）
- 手机号 + 短信验证码登录（仅预留扩展位）
- 第三方账号（微信 / Google 等）登录
- 总结结果的分享、导出功能
- 多语言总结
- 总结内容的二次编辑/笔记功能
