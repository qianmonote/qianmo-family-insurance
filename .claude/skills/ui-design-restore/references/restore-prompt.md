# 可复用提示词模板（restore-prompt）

供 `/qm:ai` 前端任务直接引用。`{{...}}` 为占位符，使用前替换。

## 实现阶段提示词

> 请根据我的 HTML+CSS 设计稿（`{{设计稿路径/链接}}`），使用 React 和 Tailwind CSS 还原设计稿的 UI 界面，注意：
>
> 1. **结构与层次**：注意 UI 组件的结构和层次，语义化拆分为可复用组件，确保还原设计稿的布局和样式。
> 2. **Tailwind 实用类**：使用 Tailwind CSS 的实用类实现样式；颜色 / 间距 / 字号 / 圆角 / 阴影等设计 token 收敛到 `tailwind.config`，确保一致性和可维护性，避免散落魔法值。
> 3. **细节与交互**：还原 hover / focus / active / disabled / loading 等状态及交互效果，确保用户体验与设计稿一致。
> 4. **动态与动画**：如设计稿中有动态交互或动画效果，请使用 React 的状态管理和生命周期方法（`useState` / `useEffect` / 过渡库）实现。
>
> 完成实现后**必须**执行像素级验证闭环（见 `verify-loop.md`）：`/goal` 量化验收目标 → MCP 多模态视觉比对（输出语义化差异清单）→ BackstopJS 视觉回归 → 按差异迭代修正，直到所有断点 × 交互态 × 动画的 diff ≤ 阈值才退出。**不得只截一张图就判通过。**

## 验证阶段提示词（驱动 MCP 视觉比对）

> 这是设计稿原图（`{{设计稿截图}}`）与我当前实现的截图（`{{实现截图}}`）。请逐项比对并输出**语义化差异清单**，按 `[布局] / [颜色] / [字体] / [间距] / [圆角] / [阴影] / [其他]` 分类，每条标注偏差量与 `阻断 / 可接受`。不要只给一个相似度分数。最后给出"要达到像素级一致，下一步该改哪些 Tailwind 类 / token"的具体修正建议。

## 迭代阶段提示词

> 依据上一轮差异清单中的**阻断项**，定向修正代码（优先调 `tailwind.config` 的 token，再调组件结构与类），然后重新截图并回到验证阶段比对。重复直到差异清单无阻断项且各维度 diff ≤ `{{阈值}}`。
