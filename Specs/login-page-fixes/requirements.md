# 登录页修复 · Requirements

## Future 功能模块

### 模块 F1：登录页测试账号预填
- **目标**：开发/演示时无需手动输入即可快速登录。
- **关键能力**：登录表单默认填入种子测试账号（`demo@qianmo.family` / `Qianmo123456`，来源 `packages/db/src/seed.ts`）。
- **依赖**：无。

### 模块 F2：认证页切换导航修复
- **目标**：登录页「新用户？立即加入」点击可正常跳转到注册页。
- **关键能力**：登录↔注册切换按钮改为可靠的链接导航。
- **依赖**：无。

## 验收标准

### F1（预填）
- **AC-1.1**：进入 `/login`，邮箱框默认显示 `demo@qianmo.family`，密码框默认显示 `Qianmo123456`。
- **AC-1.2**：预填值满足登录表单校验（邮箱格式合法、密码≥8 位），即「立即登录」按钮初始可点击、提交不触发校验错误。
- **AC-1.3**：测试账号/密码集中定义为单一常量，与 seed 中的 mock 账号一致，便于统一维护。

### F2（导航）
- **AC-2.1**：在 `/login` 点击「新用户？立即加入」跳转到 `/register`。
- **AC-2.2**：存在合法 `redirect` 查询参数时，跳转目标透传该参数（`/register?redirect=...`）。
- **AC-2.3**：导航为真实链接（渲染为 `<a href>`），不依赖 JS 点击事件，刷新/新标签打开均有效。
- **AC-2.4**：注册页「已有账号？返回登录」对称生效（同 F2 修复方式）。

## 非功能需求
- 预填仅服务于演示/测试；安全提示：若部署到对外生产环境，建议按环境开关收敛（记录于 lessons，本次默认按 PRD 字面预填）。
